# 오늘 뭐먹지 — 기획서 (1-pager)

> 끼니·약속마다 "뭐 먹지?" 고민을 대신 정해주는 **메뉴 추천 + 먹은 메뉴 도감** 미니앱.
> 비게임(추천·기록 도구) · 익명/로컬(서버·로그인 없음) · 결정적 보상형 광고가 핵심 동선.

## 한 줄 컨셉
조건(끼니·카테고리·제외)을 고르면 오늘 먹을 메뉴를 추천해주고, "다른 메뉴 더 보기"가 자연스러운 광고 시청 게이트가 되는 결정 도우미.

## 비게임 코어 (게임 아님)
- **코어 동사 = 추천 + 기록**. 조건 입력 → 메뉴 추천 카드 생성(콘텐츠 회전, 룰렛/뽑기 아님) → 먹은 메뉴를 도감에 기록.
- 승패·점수경쟁·실시간 조작·미니게임 없음 → **비게임 카테고리**.
- 게이미피케이션(도감·연속 기록·미식가 등급)은 코어 위에 얹는 **참여 레이어**.

## 코어 루프
```
조건 선택(끼니·카테고리·제외)
      ↓
[오늘의 메뉴 추천받기]  ← 첫 추천 무료
      ↓
메뉴 추천 카드(이름·이모지·카테고리·한줄 설명)
   ├─ 📺 광고 보고 다른 메뉴 더 보기   (결정적: 추천 +1)  ← 핵심 광고 게이트
   ├─ 📺 광고 보고 자세히 보기          (결정적: 이 메뉴 상세 해금)
   └─ ✅ 이거 먹을래!  → 도감 기록 + 연속 기록 +1 + 등급 경험치
      ↓
홈 복귀(연속 기록·등급·도감 진행률 갱신) → 다음 끼니에 재방문
```

## 광고 배치 지도 (R 보상형 · I 전면 · B 배너)
| 위치 | 포맷 | 트리거(결정적) |
|---|---|---|
| 추천 결과 "다른 메뉴 더 보기" | **R** | `광고 보고 다른 메뉴 추천`(+1, 옵트인) — 우유부단할수록 자주 노출 |
| 추천 결과 "자세히 보기" | **R** | `광고 보고 메뉴 상세 해금`(맛 특징·추천 상황·곁들임 팁) |
| "이거 먹을래!" 기록 후 홈 전환 | **I** | 자연 경계, 3회당 1회(세션당 ≤1 목표) |
| 홈 화면 하단 | **B** | 카드형 1개 |

- 첫 추천은 무료(그레이스풀). 추가 추천만 보상형 → **옵트인 + 결정적**(확률↑/리롤/뽑기 없음).
- 광고 키 미설정 시 즉시 통과(브라우저 개발 안 끊김).

## 가상 보상 경제 (현금 0)
- **먹은 메뉴 도감**: 6개 카테고리(한식·중식·일식·양식·분식·카페·디저트)별 메뉴 칸 채우기. 순서대로 채우는 비랜덤 수집(사행성 표면 0).
- **연속 기록(streak)**: 하루 1회 이상 "이거 먹을래!" 기록 시 연속일 +1. 놓치면 끊김(2차 기회는 추후).
- **미식가 등급**: 누적 기록 수로 레벨업(입맛 새내기 → 입맛 탐험가 → 미식가 → 대식가 → 미식 마스터).
- (향후) 광고 매출 안정 후 토스포인트 전환 지점 = "주간 기록 보상". 지금은 가상만.

## 리텐션·바이럴
- **일일 리셋**: 매일(KST) 새 "오늘의 추천" 컨텍스트 + 연속 기록 판정(`getServerTime` 기준).
- **연속 기록**: 손실회피로 매일 복귀 유도.
- **공유 카드**: 오늘 추천/먹은 메뉴를 친구에게 공유(`getTossShareLink`+`share`).

## 데이터 모델 (클라이언트 전용 · localStorage)
- 서버/DB 없음. `localStorage`에 저장:
  - `collection`: 카테고리별 수집한 메뉴명 집합
  - `streak`: { count, lastEatenDateKST }
  - `totalEaten`: 누적 기록 수(등급 산정)
- 메뉴 데이터는 앱에 내장(카테고리별 큐레이션 목록).

## 로그인 정책
- **로그인 없음.** 익명/로컬만 사용 → 추가 개인정보 미수집 → 연결끊기 엔드포인트 불필요.

## 심사 차별점
- 토스 네이티브(만보기·행운퀴즈·출석·소비분석)와 **메커니즘 다름**: 코어 동사가 "추천+기록"이고 출석/추첨 구조 아님.
- 금칙어 0(뽑기·룰렛·추첨·당첨 등 카피·인앱·스토어 전부 회피) — "추천"으로만 표현.
- 만남/연애/채팅 프레이밍 0. 위치 정보 미사용. 사행성 표면 0(비랜덤 도감 + 결정적 광고 보상).
